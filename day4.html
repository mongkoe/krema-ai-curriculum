    ChatGPT 데이터 분석 4단계

    코드를 짜는 게 아니라 ‘질문’한다 — 데이터 분석(Code Interpreter) 기능 활용

  • 파일 업로드 — 광고·분양 데이터(.xlsx/.csv)를 첨부
  • 맥락·목표 설명 — “이건 키워드별 광고 성과야. 효율 낮은 걸 찾고 싶어”
  • 분석·시각화 요청 — 요약·이상치·차트를 한 번에
  • 해석·후속 — “그래서 예산을 어떻게 재배분?”까지 이어 묻기
    • 핵심 전환 — 데이터 분석의 8할은 ‘질문 설계’. 무엇을 알고 싶은지가 명확하면 AI가 통계·차트를 대신 만든다. 숫자 결과는 한 번 검산.
    보안 — 고객 개인정보·기밀 데이터는 사내 정책 확인 후. 가능하면 식별정보(이름·연락처) 제거 후 업로드.

    1-1광고 데이터 탐색 — CTR·ROI 최적화

    키워드별 광고 성과에서 ‘효율’을 가려내고 예산을 재배분

    🖱️CTR (클릭률)

    노출 대비 클릭 = 클릭÷노출. 상·하위 10개 키워드로 ‘반응 좋은/나쁜 메시지’ 파악

    💰ROI (투자수익률)

    (수익−비용)÷비용. 예산을 ROI 높은 키워드로 이동 → 같은 돈으로 더 큰 성과

    ChatGPT가 생성·실행하는 분석 (pandas)# CTR 계산 후 상·하위 10개 키워드 df["CTR"] = df["클릭수"] / df["노출수"] top10 = df.sort_values("CTR", ascending=False).head(10) worst10 = df.sort_values("CTR").head(10) # ROI 계산 → 예산 재배분 우선순위 df["ROI"] = (df["매출"] - df["광고비"]) / df["광고비"] df.sort_values("ROI", ascending=False)[["키워드", "ROI"]]
    프롬프트 (코드 몰라도 OK)첨부한 광고 데이터로 키워드별 CTR을 계산해 상·하위 10개를 표로, ROI를 계산해 예산을 재배분할 우선순위 5개와 그 근거를 알려줘. 하위 키워드는 '중단/개선' 중 무엇이 좋을지 의견도.

    1-2분양 성과 데이터 시각화

    지역별 분양률·편차를 계산하고 ‘보고서용 한글 차트’를 자동 생성

    📈핵심 지표

    • 지역별 분양률(계약÷공급)
    • 표준편차 = 지역 간 성과 격차
    • 평균 대비 저성과 지역 식별

    🇰🇷한글 깨짐 주의

    파이썬 차트는 한글이 □로 깨진다 → koreanize-matplotlib 또는 폰트 지정으로 해결

    한글 차트 처리 + 시각화# 한글 폰트 깨짐 방지 (Colab/주피터) !pip install koreanize-matplotlib import koreanize_matplotlib, matplotlib.pyplot as plt 분양률 = df.groupby("지역")["분양률"].mean() print("지역 간 표준편차:", df["분양률"].std()) 분양률.sort_values().plot(kind="barh", title="지역별 평균 분양률")
    프롬프트분양 데이터로 지역별 평균 분양률과 표준편차를 구하고, 평균보다 낮은 지역을 강조한 가로 막대 차트를 한글 깨짐 없이 만들어줘. 보고서에 넣을 수 있게 제목·축 라벨 포함.

    1-3연간 분양 매출 분석 · 프로모션 기획

    프롬프트 — 분석에서 기획까지첨부한 2025년 수도권 분양 매출 데이터를 분석해줘. 1) 월별·지역별 추세와 전년 대비 증감 2) 저성과 구간(시기·지역)과 추정 원인 3) 회복을 위한 프로모션 전략 3가지(타겟·메시지·채널) 4) 각 전략의 KPI(측정 지표·목표치)를 표로

    ‘누구에게 파는가’를 구체적 인물로 — 2030 투자 입문층 vs 4050 상업시설 투자자

    구분2030 투자 입문층4050 상업시설 투자자
    관심사내 집 마련·시세차익·갭투자임대수익·자산 배분·절세
    라이프스타일모바일·SNS, 트렌드 민감안정 지향, 전문가 신뢰
    재정 특징자본 적음, 대출 의존여유 자본, 현금 흐름 중시
    선호 상품소형·역세권 아파트상가·오피스텔·수익형
    결정 요인가격·입지·대출 한도수익률·공실 리스크·브랜드
    페르소나 생성 프롬프트분양 타겟 2개 그룹(2030 투자 입문층 / 4050 상업시설 투자자)의 페르소나를 만들어줘. 각각 관심사·라이프스타일·재정 특징·선호 상품· 구매 결정 요인을 표로. 대표 인물(이름·나이·한 줄 소개)도 붙여줘.

    키워드 축2030 입문층 예4050 투자자 예
    분양 시기3기 신도시 분양일정상가 분양 시기
    지역·상권역세권 신축유동인구 상권 분석
    가격분양가 대출 한도상가 수익률 계산
    브랜드브랜드 아파트 비교시행사 신뢰도
    대출·세금생애최초 대출임대사업자 세금
    키워드 도출 프롬프트위 두 페르소나가 분양 정보를 찾을 때 쓸 검색 키워드를 축별(분양시기·지역상권·가격·브랜드·대출세금)로 각 5개씩 뽑아줘. 관심 투자 지역도 추정해서 함께. (2일차 네이버 API로 실제 검증 가능)
    연결 — 이 키워드는 광고 타게팅·블로그 주제·SEO에 그대로 활용. 2일차 네이버 데이터 수집과 결합하면 실제 수요 검증까지.

    2-3A/B 테스트 광고 문구 — 2030 (인스타·숏츠)

    같은 타겟, 두 가지 톤 — 감성 vs 실용으로 무엇이 먹히는지

    A감성·트렌디

    “퇴근길이 설레는 집, 여기서 시작” — 라이프스타일·감정 자극, 비주얼 중심

    B실용·정보 중심

    “역세권 3분 · 생애최초 대출 OK” — 혜택·숫자·조건 명시

    A/B 카피 생성 프롬프트2030 입문층 대상 인스타/유튜브 숏츠 광고 문구를 A/B로 만들어줘. A안: 감성·트렌디(라이프스타일 소구) B안: 실용·정보(혜택·조건 명시) 각 3개씩, 후킹 첫 문장 + 본문 + CTA. 숏츠 자막용으로 짧게.
    A/B 원칙 — 한 번에 하나의 변수만 바꿔 비교(톤만 다르게). 둘 다 좋으면 채널·시간대로 다시 쪼개 테스트.

    2-4A/B 테스트 광고 문구 — 4050 (페북·블로그)

    투자자에겐 ‘수익·안정’과 ‘브랜드·희소성’ 중 무엇이 통하나

    A수익·안정성

    “연 OO% 기대수익 · 공실 걱정 없는 입지” — 숫자·리스크 관리 강조

    B브랜드 신뢰·희소성

    “검증된 시행사 · 단 OO세대 한정” — 신뢰·희소성 소구

    A/B 카피 생성 프롬프트4050 상업시설 투자자 대상 페이스북/네이버 블로그 광고를 A/B로. A안: 수익·안정성(수익률·공실 리스크 관리) B안: 브랜드 신뢰·희소성(시행사·한정 물량) 페북용 짧은 카피 3개 + 블로그용 제목 3개. 과장·허위 표현 배제.
    법적 주의 — 수익률·확정 수익 표현은 과장·허위 광고 리스크. “기대”, “추정”을 명시하고 근거·면책 병기.

    2-5광고 반응 시뮬레이션

    집행 전, AI로 ‘예상 반응’을 미리 받아 카피를 다듬는다

    👍긍정 반응 예

    • “대출 조건이 명확해서 좋다”
    • “숫자가 구체적이라 신뢰”

    👎부정 반응 예

    • “너무 광고 같다/과장”
    • “정작 분양가가 안 보임”
    반응 시뮬레이션 프롬프트위 A/B 광고 문구 각각에 대해, 타겟 페르소나가 보일 법한 긍정/부정 반응을 3개씩 예측하고 '왜 그렇게 느끼는지' 이유를 분석해줘. 이를 토대로 더 나은 개선안 1개씩 제안해줘.
    섹션 2 정리 — 페르소나 → 키워드 → A/B 카피 → 반응 시뮬. 집행 전에 ‘가설→검증’을 AI로 빠르게 돌려 실패 비용을 줄인다.

    4-1AI 발전 현황 — 3년의 진화

    2023 → 2025 → 2026, 협업 도구가 어떻게 변했나

    시기AI의 역할사용 방식
    2023대화형 도우미물어보면 답하는 ‘챗봇’
    2025도구 사용 비서검색·코드·이미지·파일 처리

    핵심 — Agent는 ‘사람을 대체’가 아니라 ‘반복·수집·정리’를 위임하고, 사람은 판단·전략에 집중. 같은 일을 16배 빠르게.

  • Stitch가 레이아웃·카피 자리 포함 EDM 생성
  • 브랜드 색·문구 다듬어 완성

    핵심 — “같은 디자인 시스템으로”를 명시해야 4종이 따로 놀지 않는다. 5일차 디자인 시스템 개념과 연결.

    5-4v0 활용 사례

    디자인을 넘어 ‘작동하는 화면’과 문서까지

    🧪프로토타입

    클릭·입력이 되는 실제 작동 화면

    👀UX/UI 테스트

    이해관계자에게 보여주고 피드백

    📄문서 제작

    PPT·랜딩 등 다양한 산출물

    v0 프롬프트 예시분양 상담 예약 폼이 있는 랜딩 페이지를 만들어줘. - 이름·연락처·관심 평형 입력, 제출 버튼 - 제출하면 '신청 완료' 메시지 - 반응형, 깔끔한 카드 UI
    차이 — Stitch=‘디자인(보이는 것)’, v0=‘작동(되는 것)’. 정적 시안이 아니라 눌러볼 수 있는 시제품.

    5-5Stitch 결과물 → v0 프로토타입 변환

    디자인(Stitch)을 작동 프로토타입(v0)으로 이어 붙이는 실습

  • Stitch에서 만든 화면(HTML/CSS) 8개 파일 추출
  • v0에 일괄 입력(붙여넣기/업로드)
  • 모바일 뷰로 전환·정렬
  • 작동 화면(네비게이션·폼) 생성·확인
    • Stitch 디자인→ 파일 추출(8개)→ v0 입력→ 작동 프로토타입
    왜 변환? — 디자인만으론 ‘보여주기’ 한계. v0로 옮기면 실제 눌러보는 데모가 되어 컨펌·테스트가 쉬워진다.

    5-6Vercel 즉시 배포

    버튼 하나로 인터넷에 공개 — 공유 URL 획득

  • v0 프로토타입에서 Publish(배포) 클릭
  • Vercel이 자동 빌드·배포
  • 발급된 공개 URL 확인
  • 링크 공유 → 누구나 접속
    • 의미 — 이 강의 대시보드처럼, ‘로컬 → 공개 웹’이 클릭 한 번. 기획자가 아이디어를 ‘살아있는 링크’로 즉시 증명한다.
    연결 — 8일차에선 Claude Code로 만든 대시보드를 GitHub·Vercel에 배포(지금 이 사이트가 그 결과물). 오늘은 그 ‘맛보기’.
    섹션 5 정리 — Stitch(디자인) → v0(작동) → Vercel(배포). 비개발자가 ‘아이디어→공개 웹앱’을 하루 만에.
    PART 2 · 비개발자 Agent 제작

    Google AI Studio로 나만의 Agent 제작

    기획 → 빌드 → 스프레드시트 연동 → 배포까지, 실제 작동하는 업무 Agent (6-1 ~ 6-5)

    6-1 Gemini로 기획 6-2 AI Studio Build 6-3 Apps Script 코드 6-4 배포·권한 6-5 연동·고도화·공유

    6-1Gemini로 Agent 기획 (영수증 OCR 사례)

    만들기 전에 ‘설계’ — 서비스명·정의·사용자 흐름·화면 스타일을 단계별로

  • 서비스명·정의 — “영수증 정리 도우미: 사진→경비 시트 자동 입력”
  • 사용자 행동 흐름 — 영수증 촬영 → 업로드 → OCR → 시트 기록
  • 화면 스타일 — 단순·모바일, 큰 업로드 버튼
  • 필요 기능 정의 — OCR·날짜/금액 추출·시트 연동
    • Gemini 기획 프롬프트비개발자용 업무 Agent를 기획해줘. 아이디어: '영수증 사진을 올리면 날짜·상호·금액을 인식해 구글 시트 경비 장부에 자동 기록'. ① 서비스명·한 줄 정의 ② 사용자 행동 흐름(단계) ③ 화면 구성·스타일 ④ 필요한 기능 목록 ⑤ 데이터 구조(시트 열)
    핵심 — 좋은 Agent는 ‘좋은 기획’에서. 흐름·데이터 구조를 먼저 정하면 빌드가 빨라진다.

    6-2Google AI Studio — Build

    기획안을 입력하면 Gemini가 작동하는 앱을 생성

  • aistudio.google.com ▸ Build 진입
  • 모델 선택(Gemini 3 Pro Preview 등)
  • 기획안(6-1)을 프롬프트로 입력 → 앱 생성
  • 결과물 저장, 프롬프트/Annotate로 수정

    • ✏️Annotate 수정

    화면의 특정 부분을 짚어 “여기를 이렇게” 직접 지시

    💾버전 관리

    잘 된 상태를 저장하고, 실험 후 되돌리기

    Tip — 한 번에 완성 X. 작은 단위로 만들고 → 확인 → 다음 기능. 안 되면 에러를 그대로 붙여 되묻기(2일차 원리 동일).

    6-3Apps Script 코드 생성 (시트 연동)

    Agent가 만든 코드를 구글 시트에 붙여 ‘자동 기록’을 연결

    Google Apps Script (자동 생성 예시)// 웹 앱이 받은 영수증 데이터를 시트에 한 줄 추가 function doPost(e) { const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("경비"); const data = JSON.parse(e.postData.contents); sheet.appendRow([new Date(), data.상호, data.금액, data.항목]); return ContentService.createTextOutput( JSON.stringify({ result: "ok" })); }
  • 구글 시트 ▸ 확장 프로그램 ▸ Apps Script
  • 빈 편집기에 Agent가 준 코드 붙여넣기
  • 저장 후 다음 단계(배포)로
    • 안심 — 코드 한 줄도 직접 안 짠다. Agent가 생성 → 복사 → 붙여넣기. 오류 시 메시지를 그대로 Agent에 되묻는다.

    6-4웹 앱 배포 · 권한 설정

    Apps Script를 ‘웹 앱’으로 배포하고 접근 권한을 설정

  • Apps Script ▸ 배포 ▸ 새 배포 ▸ 웹 앱
  • 실행 계정: ‘나’(소유자)로
  • 액세스 범위: 용도에 맞게(나만 / 도메인 / 모두)
  • 권한 승인 — ‘미검증 앱’ 경고 시 고급 ▸ 계속 진행
  • 배포 완료 → 웹 앱 URL 발급

    • ⚠️‘미검증 앱’ 경고

    내가 만든 개인 스크립트라 뜨는 정상 경고. 출처가 본인일 때만 진행. 타인 스크립트엔 신중히.

    보안 — 액세스 범위를 넓게(‘모두’) 두면 누구나 데이터에 쓸 수 있다. 내부용은 ‘나/도메인’으로 최소화.

    6-5URL 연동 · 기능 고도화 · 공유

    웹 앱 URL을 Agent에 연결해 ‘끝까지 작동’하게 만들고 공유

    웹 앱 URL→ Agent에 연동→ 기능 보완→ Public 공유
  • 발급된 웹 앱 URL을 Agent(앱)에 입력·연결
  • 전체 흐름 테스트(영수증→인식→시트 기록)
  • 추가 기능 보완(카테고리 자동 분류 등)
  • Public 링크 활성화 → 팀과 공유
    • 섹션 6 정리 — 기획(Gemini) → 빌드(AI Studio) → 코드(Apps Script) → 배포·권한 → 연동·공유. 코딩 0으로 ‘진짜 작동하는 업무 Agent’를 완성·배포했다.
    PART 2 · 비개발자 Agent 제작
    07

    Agent를 안전하고 지속가능하게 활용하기

    만드는 것만큼 중요한 ‘잘 쓰는 법’ — 설계 원칙·윤리·프롬프트 자산화 (7-1 ~ 7-4)

    7-1 설계 3원칙 7-2 저작권·윤리 7-3 프롬프트 자산화 7-4 팀 라이브러리

    7-1성공적인 Agent 설계 3원칙

    멋진 Agent보다 ‘오래 쓰이는’ Agent

    🎯① 목적 중심

    ‘무엇을 위해’가 명확. 기능을 욕심내지 말고 한 문제를 확실히 해결

    💸② 운영 비용 이해

    API 호출·토큰엔 비용. 사용량·호출 빈도를 예측·관리

    🔐③ 보안 고려

    API Key·개인정보 노출 차단. 권한 최소화·키 분리

    현실 점검 — “돼요?”보다 “계속 쓸 수 있어요?”. 비용이 새거나 키가 노출되면 좋은 Agent도 중단된다.
    키 관리 체크 — 코드에 키 하드코딩 ✗ · 공개 저장소 업로드 ✗ · 액세스 범위 최소화 · 미사용 키 삭제(2일차 원칙 동일).

    7-2생성 AI 저작권 · 사용 윤리

    합법·윤리적으로 — 특히 부동산 ‘고객 데이터’를 다룰 때

    ⚖️저작권·사용

    • 도구별 상업 사용 라이선스 확인
    • 동의 기반 데이터만 사용
    • 인용·생성물 출처 표시

    🛡️개인정보 4원칙

    • 최소 수집(필요한 것만)
    • 목적 내 사용
    • 안전 보관(암호화·접근통제)
    • 보유기간 후 파기
    실무 기준 — 고객 명단·연락처·계약 정보는 식별정보 제거 후 또는 사내 승인된 환경에서만. ‘편해서’ 외부 AI에 그대로 붙여넣지 않는다.
    광고 윤리 — 분양 수익률·확정 표현은 과장·허위 광고 리스크. ‘추정·기대’ 명시 + 근거·면책 병기.

    7-3프롬프트 관리 · 공유의 가치

    잘 만든 프롬프트는 ‘일회용’이 아니라 ‘자산’이다

    📈왜 자산화?

    • 사용법 습득 가속
    • 답변 일관성 확보
    • 지속적 개선(버전업)
    • 조직 내 공동 학습

    🗃️무엇을 저장?

    • 잘 작동한 프롬프트 원문
    • 용도·입력 예시·주의점
    • 개선 이력(왜 바꿨나)
    핵심 — 한 사람이 찾아낸 ‘좋은 질문’을 팀 전체가 재사용하면, 개인의 노하우가 조직의 역량이 된다. 오늘 만든 모든 프롬프트가 후보.

    7-4팀 단위 프롬프트 라이브러리

    흩어진 프롬프트를 한 곳에 — 카테고리별로 누적·공유

    카테고리프롬프트 예 (오늘 자료에서)
    데이터 분석CTR·ROI 분석 / 한글 차트 시각화
    타겟·카피페르소나 / A/B 광고 문구 / 반응 시뮬
    보고서3단 구조 본문 / 경쟁 비교 / 로드맵
    Agent 제작Agent 기획 / Apps Script 연동
  • 공유 문서·노션·시트에 카테고리별 템플릿 누적
  • 각 프롬프트에 용도·예시·주의 메모
  • 주간 베스트 프롬프트 공유·표준화
    • 섹션 7 정리 — 설계 3원칙 → 윤리·보안 → 프롬프트 자산화 → 팀 라이브러리. ‘만들고 끝’이 아니라 ‘안전하게, 함께, 오래’가 진짜 생산성.
    WRAP-UP

    오늘 핵심 7가지

    • 데이터 분석 — 코드 대신 ‘질문’으로 CTR·ROI·분양성과 분석, 한글 차트까지
    • 타겟 마케팅 — 페르소나 → 키워드 → A/B 카피 → 반응 시뮬로 가설 검증
    • 분양 보고서 — 도입·경쟁·페르소나·본문(3단 구조)·로드맵으로 한 편 완성
    • Agent 이해 — ‘자율 실행 AI’, 3인 8시간 → 1인 30분
    • 바이브 코딩 — Stitch(디자인) → v0(작동) → Vercel(배포)
    • AI Studio Agent — 기획→빌드→Apps Script→배포로 진짜 작동 Agent
    • 안전 운영 — 설계 3원칙·윤리·보안·프롬프트 자산화
    관통하는 원리 — 오전엔 AI를 ‘분석가’로, 오후엔 ‘개발자’로 썼다. 비개발자가 분석부터 자동화 도구 제작·배포까지 혼자 해내는 날.

    바로 쓰는 치트시트

    데이터 분석 위임첨부 데이터로 (지표: CTR/ROI/분양률) 계산해 상·하위·이상치를 표로, 한글 깨짐 없는 차트로, 그리고 '그래서 무엇을 할지' 전략·KPI까지 제안해줘.
