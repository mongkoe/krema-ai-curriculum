    유형예시위험
    가짜 법령 조문없는 조항·번호 인용법적 오안내

    핵심 인식 — AI는 ‘모른다’ 대신 ‘그럴듯하게 지어낸다’. 특히 숫자·법령·고유명사가 위험. 외부에 나가는 정보는 반드시 사람이 검증.

    프롬프트에 ‘제약’을 걸어 거짓을 줄인다 (G-1 6원칙)

    ✅6원칙

    • ① 확실한 것만 답하기
    • ② 모르면 “모른다” 명시
    • ③ 출처·근거 함께 제시
    • ④ 추측은 ‘추정’ 표시
    • ⑤ 최신성·시점 명시
    • ⑥ 숫자·법령은 원문 확인 권고

    🎯효과

    제약을 명시하면 AI가 ‘지어내기’보다 ‘유보·표시’를 택한다. 검증 대상이 줄어든다.

    출처 강제 프롬프트 (복붙용)다음 규칙을 지켜 답해줘. 1) 확실한 사실만, 불확실하면 "확인 필요"로 표시 2) 모르면 모른다고, 지어내지 말 것 3) 각 주장에 출처(기관·문서·URL) 표기 4) 추측이 포함되면 어디가 추측인지 명시 5) 정보의 기준 시점을 적을 것 6) 숫자·법령·청약요건은 공식 원문 확인을 권고

    3-4PSK — Problem · Solution · Key result 3단 구조

    분석을 ‘설득되는 보고서’로 — 위기 진단 → 해결책 → 정량 성과

    PProblem

    위기·문제 진단 — “미분양 OO%, 경쟁 심화”

    SSolution

    패러다임 전환 해결책 — “타겟 재정의 + 채널 재배분”

    KKey result

    정량 KPI — “방문예약 +30%, 계약전환 +5%p”

    왜 PSK? — 데이터 나열은 보고가 아니다. 문제→해결→측정가능한 성과로 엮어야 의사결정자가 ‘설득’된다. 분양 기획에 그대로 적용.
    4일차 3단 구조와 차이 — 4일차(분석→해석→시사점)는 ‘본문 서술’ 구조, PSK는 ‘보고서 전체’의 설득 구조. 둘을 겹쳐 쓰면 강력.

    Claude·ChatGPT용 — 1페이지 보고서 자동 생성

    PSK 1페이지 보고서 프롬프트너는 부동산 분양 전략가다. 아래 자료를 'PSK 구조'로 1페이지 보고서로. [자료] (분석 결과·데이터 첨부) [구조] P(Problem): 핵심 문제 1~2개를 데이터로 진단 S(Solution): 문제별 전략적 해결책(왜 효과적인지 근거) K(Key result): 각 해결책의 정량 KPI와 목표치(표) [규칙] 결론부터, 출처/추정 표시, 임원 보고 톤, 1페이지 분량
    활용 — 섹션1~2에서 모은 리서치를 이 템플릿에 넣으면 ‘조사 → 설득 보고서’가 한 번에. 프롬프트 라이브러리(4일차)에 저장 1순위.

    3-6NotebookLM × PSK 결합

    ‘내 자료 기반(환각 최소)’ + ‘설득 구조(PSK)’ = 신뢰 가는 보고서

    자료 업로드(NotebookLM)→ 근거 추출·요약→ PSK 구조로 재구성→ 보고서 초안
    결합 프롬프트 (NotebookLM에서)업로드한 자료(시장보고서·실거래·공고)에 근거해서만, PSK 구조로 분양 기획 보고서 초안을 써줘. - P: 자료에서 도출한 문제(인용 표기) - S: 자료 근거 기반 해결책 - K: 측정 가능한 KPI 자료에 없는 내용은 '자료에 없음'으로 표시.
    섹션 3 정리 — ① 6유형을 알고 ② 6원칙으로 막고 ③ 교차검증으로 확인 ④ PSK로 설득. ‘빠르되 틀리지 않는’ 리서치가 PART 1의 핵심 역량.

    • “특공 일정 언제?”
    • 근거 인용과 함께 답변
    상담 응대 프롬프트너는 이 단지의 청약 상담사다. 업로드된 자료에 근거해서만 답하고, 근거 위치를 표시해. 자료에 없으면 "공식 확인 필요"라고 안내. 고객 질문: "39세 무주택, 생애최초 특별공급 가능한가요?"
    주의 — 최종 자격·당첨은 청약Home 공식 기준. 상담 노트북은 ‘1차 안내’, 확정은 공식 확인 안내를 항상 병기.

    4-4실습 3 — 시장 분석 보고서 비교

  • 공통/상이 지점 비교 요청
  • 우리 사업지 관점 시사점 도출
    • 프롬프트업로드한 3개 기관 보고서를 비교해줘. ① 공통적으로 말하는 전망 ② 기관별로 엇갈리는 지점과 이유 ③ 수치 차이(가격·공급·금리 전망)를 표로 ④ 우리 분양 사업에 주는 시사점. 각 항목 출처(기관) 표기.
    강점 — 한 기관만 보면 편향된다. 여러 보고서를 교차하면 ‘합의된 전망’과 ‘이견’이 보여 더 균형 잡힌 판단.

    효과 — 받아쓰기·정리에 쓰던 시간을 0으로. 회의 직후 ‘정리된 회의록’이 자동으로. (1일차 회의록 실습의 고도화)

    5-2좋은 Gem 지침 작성법 — TCR 3요소

    Task(역할) · Context(배경) · Reference(참고자료)

    TTask

    무엇을 하는 비서인가 — 역할·임무·출력 형식

    CContext

    배경 — 회사·단지·타겟·규칙·말투

    RReference

    Gem 지침 예시 (TCR)# Task 너는 OO단지 청약 상담 비서다. 고객 질문에 정확·친절하게 답한다. # Context - 타겟: 3040 실수요 / 톤: 신뢰감 있게, 결론부터 - 확정 정보는 청약Home 확인을 항상 안내 # Reference - 첨부된 모집공고·청약 FAQ·시세 자료 '안에서만' 답한다 - 자료에 없으면 "공식 확인 필요"로 안내
    핵심 — 1·2일차의 ‘역할·맥락·형식’ 원리와 동일. Gem은 이 지침을 고정 저장해 매번 반복하지 않는다.

    권한 — 연동 시 데이터 접근 권한을 부여하게 된다. 범위를 확인하고, 민감 메일·시트는 신중히.

    5-4실습 1 — 청약 상담 Gem

    FAQ·청약 자격·시세를 통합해 1:1 상담 자동 응대

    Gem 지침 핵심[Task] 청약 상담 응대 / [Context] 친절·정확, 확정은 공식 확인 안내 [Reference] 모집공고·청약 FAQ·주변 시세 - 답변은 '결론 → 근거(인용) → 다음 안내' 순 - 자격 판단은 '일반적 기준'임을 명시

    분양 마케팅 엑셀을 넣으면 차트·인사이트 자동 생성

    Gem 지침[Task] 마케팅 데이터를 분석해 차트와 인사이트를 만든다. [Context] 부동산 분양 마케팅(광고 성과·방문·계약 데이터) [출력] ① 핵심 지표 요약 ② 추세·이상치 ③ 보고서용 차트 ④ '그래서 무엇을' 액션 3가지. 한글 라벨, 깨짐 없이.

    📊입력

    광고비·노출·클릭·방문·계약 시트

    💡출력

    KPI·추세 차트·예산 재배분 제안

    연결 — 4일차 데이터 분석을 ‘매번 프롬프트’ 대신 전용 Gem으로. 시트만 바꿔 넣으면 끝.

    5-6실습 3 — 경쟁 단지 비교 Gem

    자사 vs 경쟁을 4축(입지·상품·가격·타겟)으로 자동 비교표

    Gem 지침[Task] 입력된 자사·경쟁 단지 정보를 4축으로 비교한다. [축] 입지 / 상품(평면·옵션) / 가격(3.3㎡당) / 타겟 [출력] ① 4축 비교표 ② 자사 우위/열위 ③ '이길 수 있는 포인트' ④ 차별화 메시지 제안. 근거 표시, 추정은 표시.
    자동화 가치 — 신규 경쟁 단지가 뜰 때마다 정보만 넣으면 비교표 완성. 4일차 ‘경쟁 비교’를 반복 가능한 Gem으로.
    입력 소스 — 6-2(AI Studio JSON 추출)로 경쟁 홈페이지를 구조화 데이터로 만들어 이 Gem에 넣으면 완전 자동.
