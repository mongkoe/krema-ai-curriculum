    클로드 Cowork 개념

    일하는 AI 작업공간 — Cowork와 Code의 차이

    Cowork란 Cowork vs Code 활용 사례 Code 탭의 역할

    Cowork란 무엇인가

    채팅을 넘어 ‘함께 일하는’ AI 작업공간

    💬일반 채팅

    묻고 답하는 대화. 결과는 ‘말’로 받고, 실행은 내가

    🧑‍💻Cowork

    AI가 도구를 직접 쓰며 실제 작업을 수행하는 공간. 파일·앱·일정을 다룬다

    • ‘작업’ 중심 — 대화가 아니라 ‘끝낸 결과물’이 목표
    • 도구 연결(MCP) — 6일차 MCP·커넥터를 기반으로 실제 일을 처리
    • 비개발자 친화 — 코드 없이 ‘시키면’ 된다
    한 줄 요약 — Cowork = “말로 시키면 실제로 일하는 AI 작업실”. 6일차가 ‘도구 붙이기’였다면, 오늘은 ‘그 도구로 본격적으로 일 시키기’.

    Cowork vs Code — 무엇이 다른가

    두 작업 탭의 역할 분담

    구분CoworkCode
    주 대상일반 업무(문서·일정·파일)코드·개발 작업
    사용자비개발자·실무자개발·기술 작업자
    방식자연어로 ‘업무’ 지시코드 작성·실행·수정
    예시“일정 정리해 메일 써줘”“이 대시보드 만들어줘”(8일차)
    핵심 — Cowork는 ‘업무 비서’, Code는 ‘개발 파트너’. 부동산 마케터는 주로 Cowork로 업무를, 필요하면 Code로 대시보드(8일차)까지.

    Cowork 활용 사례 + Code 탭의 역할

    무엇을 시킬 수 있나

    📅일정·브리핑

    캘린더 정리·하루 브리핑·미팅 메일

    📊데이터 기록

    엑셀·시트에 매물·상담 자동 기록

    🗂️파일 정리

    폴더 정리·이름 규칙·분류

    📄문서 생성

    보고서·제안서를 파일(docx·pptx)로(Skills, 섹션3~5)

    ⌨️Code 탭의 역할

    복잡한 자동화·대시보드·앱은 Code에서. 결과를 Cowork 업무에 활용

    섹션 1 정리 — Cowork는 ‘실무 자동화 작업실’. 오늘 PART 1에서 일정·엑셀·폴더를, PART 2에서 문서 생성(Skills)을 직접 시켜본다.
    PART 1 · Claude Cowork
    02

    Cowork 기초 자동화 실습 3종

    일정 보고 · 엑셀 기록 · 폴더 정리 — 매일 하는 일을 AI에게

    2-1 구글 캘린더 브리핑 2-2 엑셀 매물 시트 기록 2-3 다운로드 폴더 정리

    2-1구글 캘린더 일일 브리핑

    아침마다 ‘오늘 할 일’을 정리해 받는다

    캘린더 조회→ 일정 요약→ 브리핑 정리
    Cowork 프롬프트구글 캘린더에서 오늘 일정을 가져와서 일일 브리핑을 만들어줘. ① 시간순 일정 표 ② 미팅별 준비물·메모 ③ 이동 동선 고려한 주의사항 ④ 오늘의 우선순위 Top 3 간결하게, 출근길에 읽기 좋게.
    활용 — 매일 아침 자동 실행(스케줄)으로 발전시키면, 출근 전 ‘오늘의 브리핑’이 메일로. 분양 현장·상담 일정 관리에 유용.

    2-2엑셀 매물 시트 기록

    흩어진 매물 정보를 ‘말로 불러주면’ 시트에 정리

    Cowork 프롬프트매물 정보를 엑셀 '매물관리' 시트에 한 행으로 추가해줘. - 단지명: OO자이 / 평형: 84A / 동·호: 101동 1502호 - 매매가: 12.5억 / 상태: 매물확보 / 메모: 남향·즉시입주 열이 없으면 헤더부터 만들고, 입력일자도 자동으로.

    📥입력

    말·메모·메일 내용을 그대로

    📊출력

    구조화된 시트 한 행(자동 분류·날짜)

    응용 — “이 상담 메일 내용을 매물 시트에 정리해줘”처럼 비정형 텍스트도 자동 구조화. 2일차 엑셀 함수와 결합하면 집계까지.

    2-3다운로드 폴더 정리

    엉망인 다운로드 폴더를 규칙대로 자동 정리 (로컬 파일 접근)

  • 파일시스템 MCP(로컬) 연결 — 내 PC 폴더 접근 권한
  • 정리 규칙을 자연어로 지시
  • 유형·날짜별 하위 폴더로 분류·이동
    • Cowork 프롬프트다운로드 폴더를 정리해줘. - 이미지(jpg·png)는 '이미지', 문서(pdf·docx·xlsx)는 '문서', 압축(zip)은 '압축' 폴더로 이동 - 분양 관련 파일(파일명에 '분양·청약·단지')은 '분양자료'로 - 30일 지난 파일은 '보관' 폴더로
    주의 — 파일 이동·삭제는 되돌리기 어렵다. 먼저 “무엇을 옮길지 목록만 보여줘”로 확인 후 실행. 중요 폴더는 백업.
    섹션 2 정리 — 일정·데이터·파일 — 매일 반복하는 잡무를 Cowork가 처리. ‘말로 시키면 실제 작업이 끝난다’를 체감하는 실습.
    PART 2 · Claude Skills
    03

    클로드 Skill 개념 + 파일 생성 기능

    문서를 ‘말’이 아니라 ‘파일’로 직접 만드는 AI

    Skill이란 Skill vs MCP vs Project 파일 생성 기능 docx 실습

    Skill이란 무엇인가

    특정 작업을 ‘잘하는 법’을 패키지로 만들어 AI에 장착

    🧩Skill

    ‘이런 작업은 이렇게 하라’는 지침 + 양식 + 예시를 묶은 패키지. 필요할 때 AI가 꺼내 쓴다

    📄파일 생성

    결과를 채팅 텍스트가 아니라 실제 파일(docx·pptx·xlsx)로 만든다

    • 반복 작업 표준화 — “우리 회사 보고서 양식으로”를 매번 설명 안 해도 됨
    • 재사용·공유 — 한 번 만들면 팀 전체가 같은 품질로
    • 파일 산출물 — 바로 제출·인쇄 가능한 문서로
    비유 — Skill = ‘AI에게 쥐여주는 우리 회사 업무 매뉴얼+양식’. 4·6일차의 ‘프롬프트 자산화’를 한 단계 끌어올린 것.

    Skill vs MCP vs Project — 헷갈리지 않기

    비슷해 보이지만 역할이 다르다

    구분역할비유
    Skill작업하는 ‘방법·양식’업무 매뉴얼·서식
    MCP외부 ‘도구’ 연결손발·연장(6일차)
    Project‘자료·맥락’ 보관자료 바인더(1일차)
    함께 쓰면 — Project(자료) + MCP(도구) + Skill(방법) = 자료를 보고, 도구를 쓰며, 우리 양식대로 일하는 AI. 오늘 Skill로 ‘방법’을 장착한다.

    실습 — 시장 분석 텍스트를 docx로

    분석 내용을 ‘제출 가능한 워드 문서’로 바로 생성

    분석 텍스트→ 문서 구조화→ docx 파일 생성
    Cowork 프롬프트아래 시장 분석 내용을 워드 문서(.docx)로 만들어줘. - 표지(제목·날짜·작성자) + 목차 - 본문: 시장 현황 / 경쟁 분석 / 시사점 (제목 스타일 적용) - 표·번호 매기기 포함, 바로 제출 가능한 형식 [분석 내용] (붙여넣기)
    핵심 — 채팅 텍스트를 복사·붙여 워드에 다시 꾸미던 작업이 사라진다. ‘파일로 바로 산출’. 다음 섹션에서 이걸 ‘우리 브랜드 양식’으로 고정한다.
    PART 2 · Claude Skills
    04

    브랜드 스타일 문서 생성 Skill 만들기

    ‘회사 양식’을 패키징해서, 누가 써도 같은 문서가 나오게

    왜 만드나 브랜드 가이드라인 적용 스킬로 패키징 스킬 편집

    왜 ‘브랜드 Skill’을 만드나

    매번 “우리 양식으로, 우리 색으로, 우리 톤으로”를 반복하지 않으려고

    😩Skill 없이

    • 매번 양식·색·톤 설명
    • 사람마다 결과물이 제각각
    • 브랜드 일관성 깨짐

    😎Skill 있으면

    • “보고서 만들어줘” 한 줄
    • 항상 같은 양식·색·톤
    • 팀 전체 품질 균일
    가치 — 브랜드는 ‘일관성’이 생명. Skill은 그 일관성을 자동으로 강제한다. 디자이너·편집자 없이도 브랜드 문서가 표준화.

    브랜드 가이드라인 적용

    우리 브랜드의 ‘규칙’을 Skill에 담는다

    🎨색·서체

    브랜드 컬러(HEX)·지정 폰트·제목 스타일

    🗂️레이아웃

    표지·목차·머리말/꼬리말·로고 위치

    ✍️톤·문체

    존댓말·결론 우선·과장 배제 등 글쓰기 규칙

    브랜드 가이드 (Skill 지침 예시)# 브랜드 문서 규칙 색상: 메인 #1e3a5f(네이비), 포인트 #f97316(오렌지) 서체: 제목 Pretendard Bold, 본문 Pretendard Regular 구조: 표지 → 목차 → 본문(결론 우선) → 부록 톤: 정중한 존댓말, 과장·확정 수익 표현 금지 필수: 모든 표지에 로고·작성일·문서번호
    준비물 — 회사 로고, 브랜드 컬러 코드, 기존 보고서 양식 1부. 이걸 Skill에 ‘참고자료’로 넣으면 더 정확.

    스킬로 패키징 + 편집

    규칙을 ‘Skill’로 저장하고, 계속 다듬는다

  • 지침 작성 — 브랜드 규칙 + 양식 + 예시 문서
  • Skill로 저장(패키징) — 이름·설명·트리거 지정
  • 호출·테스트 — “브랜드 보고서로 만들어줘” → 결과 확인
  • 편집·개선 — 어색한 부분을 지침에 반영해 업데이트
    • Skill 호출 예방금 만든 '브랜드 보고서' 스킬로, 이 시장 분석 내용을 우리 회사 양식 docx로 만들어줘.
    섹션 4 정리 — 브랜드 규칙을 Skill로 패키징하면 ‘한 줄로 브랜드 문서’가 완성. 한 번 만든 Skill은 팀 자산이자 영구 재사용. 다음 섹션에선 PPT까지.

    주변 시세·상권·교통·리뷰 등 웹 데이터를 자동 수집(6일차 복습)

    준비 — Apify 커넥터를 Cowork에 연결(6일차 방식). 수집 대상(지역·키워드)을 미리 정해두면 빠르다.

    사업지 입지분석

    수집한 데이터로 ‘근거 있는’ 입지 분석

    Cowork 프롬프트Apify로 수집한 'OO지역' 데이터를 바탕으로 입지 분석 보고서를 만들어줘. ① 교통(역세권·도로) ② 생활인프라(학군·상권·병원) ③ 개발 호재·리스크 ④ 주변 시세·경쟁 단지 ⑤ 종합 평가(10점)와 마케팅 셀링포인트 3가지 근거(데이터)를 함께 표시하고, 추정은 '추정'으로.

    🚇교통·접근

    역·도로·소요시간

    🏫생활인프라

    학군·상권·의료

    📈시세·호재

    주변 시세·개발 계획

    연결 — 5일차 PSK 구조·NotebookLM 분석과 결합하면 더 탄탄. 분양 검토 보고서의 ‘입지’ 파트에 직결.

    디자인 시스템 PPT 생성

    분석 결과를 ‘브랜드 디자인 시스템’이 적용된 PPT로

    Cowork 프롬프트 (Skill 활용)위 입지분석 결과를 우리 '브랜드 PPT' 스킬로 발표자료(.pptx)로 만들어줘. - 표지 + 목차 + 입지 개요 + 항목별 분석(교통/인프라/시세) + 종합평가 - 브랜드 색(네이비+오렌지), 일관된 레이아웃·아이콘 - 각 슬라이드 핵심 1줄 + 근거 데이터 시각화

    🎨디자인 시스템

    색·서체·여백·컴포넌트를 일관 적용(5일차 디자인 개념)

    📑산출물

    바로 발표·제출 가능한 .pptx

    PART 3 · 로컬 MCP 서버
    06

    로컬 MCP 개념

    리모트 vs 로컬, 무한 확장의 원리

    로컬 MCP란 리모트 vs 로컬 비교 무한 확장 원리

    로컬 MCP란?

    내 PC에 직접 설치해서 돌리는 MCP 서버

    ☁️리모트 MCP

    제공사 서버에 연결. 설치 없이 ‘제공되는 것만’ 사용(6일차 커넥터)

    💻로컬 MCP

    내 PC에 직접 설치. 전 세계 개발자가 만든 모든 MCP를 가져다 사용

    핵심 차이 — 리모트는 ‘제공되는 메뉴’만, 로컬은 ‘원하는 도구를 직접 설치’. 로컬 = 무한 확장의 열쇠. 단, 설치·관리 수고가 따른다.

    리모트 vs 로컬 MCP 비교

    구분리모트 MCP로컬 MCP
    설치불필요(연결만)내 PC에 직접 설치
    사용 가능 범위제공된 것만공개된 모든 MCP
    내 PC 접근제한적파일·프로그램 자유
    난이도쉬움설정 필요(오늘 실습)
    적합빠른 시작맞춤·확장·전문 작업
    선택 — 처음엔 리모트로 시작, 필요가 늘면 로컬로 확장. 오늘은 로컬 설치까지 배워 ‘한계 없는 도구 상자’를 갖춘다.

    무한 확장의 원리

    ‘표준 규칙(MCP)’ 덕분에, 누가 만든 도구든 끼워 쓸 수 있다

    공개된 MCP 서버→ 내 PC에 설치→ 설정 파일 등록→ Claude의 새 능력
    원리 — MCP는 ‘표준 규칙’(6일차). 그래서 누군가 새 도구를 MCP로 만들면, 나는 설치만 하면 내 AI가 그 능력을 얻는다. 도구가 늘수록 AI가 강해진다.
    섹션 6 정리 — 로컬 MCP = ‘AI 능력의 무한 확장’. 다음 섹션에서 부동산에 유용한 5개를 직접 설치한다.
    PART 3 · 로컬 MCP 서버
    07

    주요 로컬 MCP 서버 5선 + 설치 실습

    부동산 마케터에게 유용한 5개를 설치하고 조합해 쓰기

    5선 소개 설치 방법 실거래가 조회 블로그 자동생성 5개 조합 시나리오

    7-1주요 로컬 MCP 서버 5선

    부동산 마케팅에 바로 쓰는 다섯 가지

    MCP 서버기능부동산 활용
    Naver Search네이버 검색·뉴스·블로그지역·단지 정보·여론
    YouTube Transcript영상 자막 추출경쟁사 영상 분석
    Nano Banana이미지 생성·편집분양 비주얼(3·5일차)
    n8n워크플로 자동화수집·발송 자동화(6일차)
    real-estate부동산 실거래가시세·실거래 조회
    조합의 힘 — 검색(Naver) + 시세(real-estate) + 이미지(Nano Banana) + 자동화(n8n)를 한 AI에 모으면, ‘조사→분석→비주얼→발송’이 한 자리에서.

    7-2설치 방법

    설정 파일에 ‘서버 등록’ — 한 번 해두면 끝

  • Node.js 설치 — 대부분의 로컬 MCP 실행 환경
  • 설정 파일 열기 — Claude 데스크탑 ▸ 설정 ▸ MCP 설정(JSON)
  • 서버 등록 — 아래처럼 명령·키 입력
  • 재시작·확인 — Claude 재시작 후 도구 목록에서 확인
    • MCP 설정 파일 (claude_desktop_config.json){ "mcpServers": { "naver-search": { "command": "npx", "args": ["-y", "@modelcontextprotocol/naver-search"], "env": { "NAVER_CLIENT_ID": "발급ID", "NAVER_CLIENT_SECRET": "발급KEY" } } } }
    주의 — 설정 파일은 정확한 JSON 형식이어야 한다(쉼표·괄호). 막히면 에러를 그대로 Claude에 붙여 물어보면 고쳐준다. API 키는 절대 공개 금지.

    7-3활용 실습 ➊ 관심 단지 실거래가 조회

    real-estate MCP로 시세를 ‘대화로’ 조회

    real-estate MCP→ 실거래 조회→ 표·분석
    실습 프롬프트'강남구 OO아파트 84㎡'의 최근 1년 실거래가를 조회해서 ① 거래 내역 표(날짜·층·금액) ② 평균/최고/최저 ③ 가격 추세와 우리 분양가 책정 시사점을 정리해줘.
    편의 — 사이트를 일일이 검색·복붙하지 않고 대화로 즉시. 5일차 NotebookLM·6일차 공공데이터와 함께 ‘시세 분석 3종 세트’.

    7-4활용 실습 ➋ 네이버 검색 → 블로그 글 자동 생성

    Naver Search MCP로 조사하고, 바로 블로그 글까지

    Naver Search→ 지역·단지 조사→ 블로그 글 작성
    실습 프롬프트네이버 검색으로 'OO지역 분양·생활인프라' 최신 정보를 조사하고, 그 내용을 바탕으로 블로그 글을 써줘. - 제목(클릭 유도) + 소제목 구조 - 실수요자 관점, 정보형, 과장 없이 - 마지막에 분양 안내·문의 유도 (광고 표기)
    주의 — 검색 결과의 사실(시세·정책)은 검증 필요(5일차 할루시네이션). 광고성 글은 ‘광고’ 표시 등 규정 준수.

    7-5활용 실습 ➌ 5개 MCP 조합 시나리오

    5개를 모두 연결해 ‘조사→분석→비주얼→발행’ 한 흐름

    Naver(조사)→ real-estate(시세)→ YouTube(경쟁 영상)→ Nano Banana(비주얼)→ n8n(발행)
    통합 프롬프트'OO지역 신규 분양' 콘텐츠를 만들어줘. ① 네이버로 지역 정보 조사 ② 실거래가로 시세 분석 ③ 경쟁사 유튜브 영상의 핵심 포인트 정리 ④ 카드뉴스 이미지 생성 ⑤ n8n으로 블로그·SNS에 발행
    섹션 7 정리 — 로컬 MCP 5개를 조합하면 ‘리서치→시세→경쟁분석→비주얼→발행’이 한 자리에서. 나만의 부동산 마케팅 자동화 허브가 완성된다.
    WRAP-UP

    오늘 핵심 7가지

    • Cowork — ‘말로 시키면 실제로 일하는’ AI 작업공간(Code와 역할 분담)
    • Cowork 자동화 — 일정 브리핑·엑셀 기록·폴더 정리
    • Skill — 작업 ‘방법·양식’을 패키지로, 문서를 파일로 생성
    • 브랜드 Skill — 회사 양식을 패키징해 일관된 문서 자동화
    • 입지분석 PPT — Apify→분석→디자인 PPT 한 흐름
    • 로컬 MCP — 내 PC에 설치해 AI 능력을 무한 확장
    • 5선 조합 — 검색·시세·영상·이미지·자동화를 한 허브로
    관통하는 원리 — “Cowork로 일하고, Skill로 표준화하고, 로컬 MCP로 확장한다.” 6일차(도구 붙이기)에서 오늘은 ‘본격적으로 일 시키고, 나만의 도구함을 짓는’ 단계로.
