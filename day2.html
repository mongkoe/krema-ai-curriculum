    01

    업무에 바로 쓰는 엑셀 함수 생성

    함수 문법을 외울 필요 없이, ChatGPT에게 “원하는 결과”를 말하면 수식을 만들어 준다 (1-1 ~ 1-7)

    1-1 IF 재고 1-2 SUMIF 1-3 VLOOKUP 1-4 COUNTIFS 1-5 AVERAGEIF 1-6 IF·TODAY 연체 1-7 IFERROR·INDEX·MATCH
    시작 전 핵심

    ChatGPT로 엑셀 함수 만드는 4단계

    오늘 모든 함수 실습에 공통으로 쓰는 패턴

  • 데이터 구조 설명 — “A열 동, B열 호수, D열 분양상태, F열 잔여세대…”
  • 원하는 결과 말하기 — “잔여세대가 5 이하면 ‘마감임박’으로 표시하고 싶어”
  • 수식 받기 → 붙여넣기 — 셀에 붙이고 Enter, 아래로 채우기
  • 안 맞으면 되묻기 — “기준을 3으로 바꿔줘”, “오류(#N/A) 처리도 추가”
    • 만능 프롬프트 틀너는 엑셀 함수 전문가다. 내 데이터 구조는 다음과 같아. [구조] A열=동, B열=호수, D열=분양상태(계약/미계약), F열=잔여세대 [원하는 것] (자연어로 설명) → 한국어 설명과 함께 '바로 붙여넣을 수식'을 제시해줘. 셀 위치 예시도.
    왜 중요한가 — 함수 이름을 몰라도 된다. “무엇을 하고 싶은지”만 정확히 말하면 AI가 함수를 골라준다.

    1-1IF 함수 — 재고/잔여세대 확인 자동화

    조건에 따라 두 갈래로 다른 값을 표시 (가장 기본이자 가장 많이 쓰는 함수)

    📌이럴 때

    • 잔여세대 임계치 경고
    • 목표 달성/미달 표시
    • 합격/불합격 판정

    🔤문법

    =IF(조건, 참일때, 거짓일때)

    조건이 맞으면 ‘참일때’ 값을, 아니면 ‘거짓일때’ 값을 반환

    실무 예제 — F열 잔여세대 기준# 잔여세대(F2)가 5 이하이면 "마감임박", 아니면 "여유" =IF(F2<=5, "마감임박", "여유") # 중첩: 0이면 "완판", 5이하 "마감임박", 그 외 "여유" =IF(F2=0, "완판", IF(F2<=5, "마감임박", "여유"))
    ChatGPT에게F열은 잔여세대 수야. 잔여가 0이면 '완판', 5이하면 '마감임박', 나머지는 '여유'로 G열에 표시하는 수식을 만들어줘.
    주의 — 텍스트는 큰따옴표 " "로 감싼다. 조건 비교는 <= >= <>(같지않음) 사용.

    1-2SUMIF — 특정 조건의 합계

    “특정 평형의 계약금 총합”처럼 조건에 맞는 값만 더한다

    📌이럴 때

    • 평형별 계약금 합계
    • 담당자별 매출 합계
    • 특정 월 입금 총액

    🔤문법

    =SUMIF(조건범위, 조건, 합계범위)

    조건범위에서 조건과 맞는 행의 합계범위 값을 더함

    실무 예제 — 평형(C열)별 계약금(E열) 합계# C열이 "84A"인 행들의 E열(계약금) 합계 =SUMIF(C:C, "84A", E:E) # 조건을 셀(H2)로 받으면 평형만 바꿔가며 재사용 =SUMIF(C:C, H2, E:E) # 다중 조건은 SUMIFS (조건 여러 개) =SUMIFS(E:E, C:C, "84A", D:D, "계약")
    ChatGPT에게C열은 평형, D열은 계약상태, E열은 계약금이야. '84A' 타입이면서 '계약' 완료된 건의 계약금 합계를 구하는 수식 알려줘.
    Tip — 조건이 2개 이상이면 SUMIF가 아니라 SUMIFS(합계범위가 맨 앞). 헷갈리면 AI에게 “조건 2개”라고만 말하면 됨.

    1-3VLOOKUP — 다른 표에서 데이터 불러오기

    고객ID로 이름·연락처를 ‘조회’해 오는 함수 (실무 활용 1순위)

    📌이럴 때

    • 고객ID → 이름·연락처 조회
    • 단지코드 → 분양가 매칭
    • 두 시트 데이터 합치기

    🔤문법

    =VLOOKUP(찾을값, 표범위, 가져올열번호, FALSE)

    FALSE=정확히 일치(필수). 표의 맨 왼쪽 열에서 찾는다

    실무 예제 — 고객DB에서 정보 조회# H2(고객ID)로 고객DB(A:E)에서 5번째 열(연락처)을 가져옴 =VLOOKUP(H2, 고객DB!A:E, 5, FALSE) # 없는 값일 때 오류 대신 "미등록" 표시 (1-7과 연계) =IFERROR(VLOOKUP(H2, 고객DB!A:E, 5, FALSE), "미등록") # 최신 엑셀이면 XLOOKUP 추천 (좌우 자유, 더 안전) =XLOOKUP(H2, 고객DB!A:A, 고객DB!E:E, "미등록")
    ChatGPT에게'고객DB' 시트의 A열은 고객ID, E열은 연락처야. 현재 시트 H2에 입력한 고객ID의 연락처를 가져오고, 없으면 '미등록'으로 표시하는 수식을 만들어줘.
    가장 흔한 실수 — 찾을 값이 표의 맨 왼쪽 열에 있어야 한다. 위치가 안 맞으면 AI에게 INDEX/MATCH 또는 XLOOKUP으로 요청.

    1-4COUNTIFS — 여러 조건 충족 개수

    “84A이면서 계약 완료된 세대 수”처럼 조건에 맞는 ‘건수’를 센다

    📌이럴 때

    • 평형별·상태별 계약 건수
    • 기간 내 상담 건수
    • 대시보드 KPI 카운트

    🔤문법

    =COUNTIFS(범위1, 조건1, 범위2, 조건2, …)

    모든 조건을 동시에 만족하는 행의 개수

    실무 예제 — 평형 × 상태별 건수# 평형(C열)="84A" 이고 상태(D열)="계약"인 건수 =COUNTIFS(C:C, "84A", D:D, "계약") # 날짜 범위 조건 — 5월 한 달 상담 건수 (G열=상담일) =COUNTIFS(G:G, ">="&DATE(2026,5,1), G:G, "<="&DATE(2026,5,31))
    ChatGPT에게C열 평형, D열 계약상태, G열 상담일자야. '84A'면서 '계약'인 건수, 그리고 2026년 5월 상담 건수를 각각 세는 수식 두 개를 만들어줘.
    Tip — 부등호 조건은 ">="&기준값 형태로 &(연결) 사용. 날짜는 DATE()로 만들면 안전.

    1-5AVERAGEIF — 조건부 평균

    “101동의 평균 분양가”처럼 조건에 맞는 값들의 평균

    📌이럴 때

    • 동별 평균 분양가
    • 평형별 평균 계약금
    • 채널별 평균 문의수

    🔤문법

    =AVERAGEIF(조건범위, 조건, 평균범위)

    다중 조건은 AVERAGEIFS

    실무 예제 — 동(A열)별 평균 분양가(E열)# A열이 "101동"인 행들의 E열(분양가) 평균 =AVERAGEIF(A:A, "101동", E:E) # 다중 조건: 101동 + 84A 평형의 평균 분양가 =AVERAGEIFS(E:E, A:A, "101동", C:C, "84A")
    ChatGPT에게A열 동, C열 평형, E열 분양가야. 동별 평균 분양가를 구하고, '101동 84A'의 평균도 따로 구하는 수식을 만들어줘. 빈칸은 평균에서 제외돼야 해.
    주의 — 평균범위에 빈칸/0이 섞이면 결과가 왜곡된다. “빈칸 제외”를 명시하거나 AVERAGEIFS로 조건 추가.

    1-6IF + TODAY — 연체/D-day 자동 계산

    오늘 날짜를 기준으로 ‘연체 여부’나 ‘남은 기한’을 매일 자동 갱신

    📌이럴 때

    • 중도금 납부 연체 표시
    • 계약 만료 D-day
    • 이벤트 마감 카운트

    🔤핵심 함수

    TODAY() = 오늘 날짜 (파일 열 때마다 갱신)

    DATEDIF / 날짜 뺄셈으로 일수 계산

    실무 예제 — 중도금 납부일(H열) 기준# 납부예정일(H2)이 오늘보다 지났고 미납이면 "연체" =IF(AND(H2<TODAY(), I2="미납"), "연체", "정상") # 마감까지 남은 일수 (음수면 이미 지남) =H2-TODAY() → 셀 서식을 '숫자'로 # D-day 표기: "D-7", "D-DAY", "D+3" =IF(H2=TODAY(), "D-DAY", IF(H2>TODAY(), "D-"&(H2-TODAY()), "D+"&(TODAY()-H2)))
    ChatGPT에게H열은 중도금 납부예정일, I열은 납부상태(납부/미납)야. 오늘 기준으로 예정일이 지났는데 미납이면 '연체', 아니면 '정상'으로 표시하는 수식을 만들어줘.
    주의 — 결과가 날짜(1900-01-07)로 보이면 셀 서식을 ‘일반/숫자’로 바꾼다. AI에게 “셀 서식도 알려줘”라고 요청.

    1-7IFERROR + INDEX + MATCH — 안전한 양방향 조회

    VLOOKUP의 한계(왼쪽만 조회)를 넘고, 오류까지 깔끔하게 처리하는 ‘실무 끝판왕’ 조합

    ①MATCH

    값이 몇 번째에 있는지 위치를 찾음

    ②INDEX

    그 위치의 값을 꺼냄

    ③IFERROR

    못 찾으면 대체값 표시

    실무 예제 — 고객 → 담당자 추출 (오른쪽→왼쪽도 OK)# MATCH: 고객(H2)이 고객목록(B열)에서 몇 번째 행인지 # INDEX: 담당자(A열)에서 그 행의 값을 꺼냄 # IFERROR: 없으면 "미배정" =IFERROR(INDEX(A:A, MATCH(H2, B:B, 0)), "미배정")
    ChatGPT에게A열은 담당자, B열은 고객명이야. (담당자가 고객 왼쪽에 있어서 VLOOKUP이 안 돼) H2에 입력한 고객의 담당자를 INDEX/MATCH로 찾고, 없으면 '미배정'으로 표시하는 수식을 만들어줘.
    모듈 1 정리 — 함수는 7개지만 패턴은 하나. “데이터 구조 + 원하는 결과”를 ChatGPT에 말하면 끝. 외울 건 함수 이름이 아니라 ‘질문하는 법’이다.

    개념

    VBA / 매크로란?

    엑셀에 내장된 자동화 언어 — “이 작업들을 순서대로 실행해” 라고 기록해 둔 것

    ⚙️이런 일을 자동화

    • 여러 시트를 한 번에 PDF로
    • 반복 서식·정리 작업
    • 조건에 따라 메일/파일 생성

    🧩구성 요소

    • Sub … End Sub : 매크로 한 덩어리
    • For … Next : 반복
    • If … Then : 조건 분기
    오늘의 방식 — VBA 문법을 배우는 게 목적이 아니다. “이런 매크로 만들어줘”라고 ChatGPT에 요청 → 코드 복사 → 엑셀에 붙여넣고 실행. 안 되면 오류 메시지를 그대로 붙여 되묻는다.

    2-1엑셀 VBA 기본 사용 설정

    매크로를 쓰려면 먼저 ‘개발 도구’를 켜고 편집기를 연다

  • 개발 도구 탭 표시 — 파일 ▸ 옵션 ▸ 리본 사용자 지정 ▸ ‘개발 도구’ 체크
  • 편집기 열기 — 단축키 Alt + F11 (Visual Basic Editor)
  • 모듈 추가 — 삽입(Insert) ▸ 모듈(Module) → 여기에 코드 붙여넣기
  • 실행 — F5 또는 개발도구 ▸ 매크로 ▸ 실행
  • 저장 — 파일 형식을 .xlsm(매크로 사용 통합 문서)으로

    • ⚠️보안 경고가 뜨면

    ‘콘텐츠 사용’을 눌러야 매크로가 동작. 출처 불명 파일의 매크로는 실행하지 말 것

    💾꼭 기억

    일반 .xlsx로 저장하면 매크로가 사라진다. 반드시 .xlsm

    2-2여러 시트를 PDF로 자동 저장

    단지별/월별 시트가 수십 개일 때, 한 번에 각각 PDF로 내보내기

    VBA 코드 — 모든 시트를 개별 PDF로Sub 시트별_PDF저장() Dim ws As Worksheet Dim path As String path = ThisWorkbook.path & "\PDF\" ' 저장 폴더 If Dir(path, vbDirectory) = "" Then MkDir path For Each ws In ThisWorkbook.Worksheets ws.ExportAsFixedFormat Type:=xlTypePDF, _ Filename:=path & ws.Name & ".pdf" Next ws MsgBox "PDF 저장 완료!" End Sub
    ChatGPT에게 (코드 직접 안 짜도 됨)엑셀 VBA 매크로를 만들어줘. 현재 통합문서의 '모든 시트'를 각각 별도 PDF 파일로, 파일명은 시트 이름으로, 같은 폴더의 'PDF' 하위폴더에 저장해줘. 완료되면 메시지박스도 띄워줘.
    응용 — “특정 시트만”, “파일명에 오늘 날짜 추가” 등 조건을 덧붙여 되물으면 코드를 수정해 준다.

    2-3회원 민감정보 마스킹 처리

    고객 명단을 외부 공유 전, 연락처·이름을 자동으로 가린다 (개인정보 보호)

    VBA 코드 — 연락처 가운데 4자리 마스킹Sub 연락처_마스킹() Dim rng As Range, c As Range Set rng = Range("D2:D" & Cells(Rows.Count, 4).End(xlUp).Row) For Each c In rng If Len(c.Value) >= 11 Then ' 010-1234-5678 c.Value = Left(c.Value, 3) & "-****-" & Right(c.Value, 4) End If Next c End Sub
    ChatGPT에게D열에 휴대폰 번호(010-1234-5678 형식)가 있어. 가운데 4자리를 ****로 가리는 VBA 매크로를 만들어줘. 이름(B열)도 가운데 글자를 ○로 바꾸는 버전도 같이.
    주의 — 원본을 덮어쓰므로 사본에서 실행하거나, 마스킹 결과를 옆 열에 쓰도록 요청하면 안전.

    2-4설문조사 결과 시각화

    응답 집계 → 자동으로 차트 생성까지 한 번에

    VBA 코드 — 집계표로 막대 차트 자동 생성Sub 설문_차트생성() Dim ch As ChartObject Set ch = ActiveSheet.ChartObjects.Add(Left:=300, Top:=20, _ Width:=400, Height:=250) With ch.Chart .SetSourceData Source:=Range("A1:B6") ' 항목/응답수 .ChartType = xlColumnClustered .HasTitle = True .ChartTitle.Text = "설문 결과" End With End Sub
    ChatGPT에게A1:B6에 설문 항목과 응답수가 있어. 이걸 막대그래프로 자동 생성하는 VBA 매크로를 만들어줘. 제목은 '설문 결과'로 하고, 차트는 데이터 옆에 배치해줘.
    대안 — VBA가 부담되면, 같은 데이터를 COUNTIF로 집계한 뒤 엑셀 기본 ‘추천 차트’를 써도 충분. 자동화가 필요할 때만 매크로.

    2-5월별 매출 합계 계산 및 차트 생성

    날짜별 거래 데이터 → 월별 집계 → 추세 차트까지 자동

    VBA 코드 — 월별 합계 후 꺾은선 차트Sub 월별매출_집계차트() Dim i As Integer ' 가정: A열=날짜, B열=매출. 결과는 D열(월), E열(합계) For i = 1 To 12 Cells(i + 1, 4).Value = i & "월" Cells(i + 1, 5).Value = _ Application.SumProduct((Month(Range("A2:A1000")) = i) * Range("B2:B1000")) Next i ActiveSheet.Shapes.AddChart2(227, xlLine).Select ' 꺾은선 ActiveChart.SetSourceData Source:=Range("D1:E13") End Sub
    ChatGPT에게A열은 거래날짜, B열은 매출금액이야. 1~12월 매출 합계를 D·E열에 집계하고, 월별 추세를 꺾은선 차트로 그리는 VBA 매크로를 만들어줘.
    모듈 2 정리 — 매크로 = “반복 작업의 녹화 버튼”. 코드는 ChatGPT가, 실행은 내가. 오류가 나면 메시지를 그대로 복사해 되묻는 것이 핵심 스킬.

    3-3구글 뉴스 기사 자동 수집

    Python 코드 — 구글 뉴스 RSS 수집 → 엑셀 저장import requests, pandas as pd from bs4 import BeautifulSoup 키워드 = "아파트 분양" url = f"https://news.google.com/rss/search?q={키워드}&hl=ko" res = requests.get(url) soup = BeautifulSoup(res.content, "xml") 데이터 = [] for item in soup.find_all("item")[:20]: 데이터.append({"제목": item.title.text, "링크": item.link.text, "날짜": item.pubDate.text}) df = pd.DataFrame(데이터) df.to_excel("뉴스수집.xlsx", index=False) # 엑셀로 저장 print(f"{len(df)}건 수집 완료")
    ChatGPT에게구글 뉴스에서 '아파트 분양' 키워드 최신 기사 20건의 제목·링크·날짜를 수집해서 엑셀로 저장하는 파이썬 코드를 만들어줘. 구글 Colab에서 바로 실행 가능하게.

    3-6뉴스 제목 기반 키워드 · 트렌드 분석

    수집한 제목을 모아 ‘요즘 무슨 말이 많은지’ 빈도·흐름을 분석

    Python 코드 — 핵심 키워드 빈도 Top 10from collections import Counter import re 제목들 = df["제목"].tolist() 단어 = re.findall(r"[가-힣]{2,}", " ".join(제목들)) # 2글자+ 한글 불용어 = {"기자", "종합", "속보"} 단어 = [w for w in 단어 if w not in 불용어] for 키워드, 횟수 in Counter(단어).most_common(10): print(키워드, 횟수)
    ChatGPT에게 (분석까지 위임)방금 수집한 뉴스 제목 데이터(df)에서 1) 가장 많이 등장한 키워드 Top 10을 막대그래프로 2) 날짜별 기사 수 추세를 선그래프로 그려줘. 3) 이 데이터로 읽을 수 있는 시장 트렌드 3가지도 요약해줘.
    모듈 3 정리 — 수집(크롤링) → 정리(pandas) → 분석/시각화 → 인사이트. 이 파이프라인을 한 번 만들면 키워드만 바꿔 무한 재사용한다.

    개념

    API란? 왜 크롤링보다 좋은가

    API = 서비스가 공식적으로 열어둔 ‘데이터 창구’

    🕷️크롤링

    • 화면(HTML)을 긁음 → 구조 바뀌면 깨짐
    • 차단·법적 리스크
    • 비정형 데이터

    🔌API

    • 공식 허가된 통로 → 안정적
    • 깔끔한 데이터(JSON)
    • 인증키로 합법적 사용
    비유 — 크롤링이 ‘담 넘어 구경’이라면, API는 ‘정문으로 출입증 받고 입장’. 가능하면 항상 API 먼저.

    4-1네이버 검색 API 소개 및 발급

    뉴스·블로그·쇼핑·카페 등을 검색해 결과를 데이터로 받는 무료 API

  • developers.naver.com 접속 → 네이버 로그인
  • Application ▸ 애플리케이션 등록
  • 사용 API에서 ‘검색’ 선택
  • 환경: WEB 설정 + URL 입력(예: http://localhost)
  • 발급된 Client ID + Client Secret 복사·보관

    • 🔐키 관리 주의

    Client Secret은 비밀번호다. 코드에 그대로 적어 공유/업로드하지 말 것. 하루 호출 한도(예: 검색 25,000회) 내에서 무료.

    4-2네이버 뉴스 데이터 수집

    발급받은 키로 뉴스 검색 결과를 JSON으로 받아 표로 정리

    Python 코드 — 네이버 뉴스 검색 APIimport requests, pandas as pd headers = { "X-Naver-Client-Id": "발급받은_ID", "X-Naver-Client-Secret": "발급받은_SECRET" } params = {"query": "재건축", "display": 50, "sort": "date"} url = "https://openapi.naver.com/v1/search/news.json" res = requests.get(url, headers=headers, params=params) items = res.json()["items"] df = pd.DataFrame(items)[["title", "link", "pubDate"]] df.to_excel("네이버뉴스.xlsx", index=False)
    ChatGPT에게네이버 검색 API(뉴스)로 '재건축' 키워드 최신순 50건을 수집해서 제목·링크·날짜를 엑셀로 저장하는 파이썬 코드를 만들어줘. Client ID/Secret 넣는 위치를 주석으로 표시해줘.
    핵심 — 엔드포인트만 news → blog → shop으로 바꾸면 4-3, 4-4가 된다.

    4-3네이버 블로그 데이터 수집

    블로그 후기·바이럴 흐름 파악 — 엔드포인트만 교체

    Python 코드 — 뉴스 코드에서 한 줄만 변경# 뉴스: .../search/news.json url = "https://openapi.naver.com/v1/search/blog.json" params = {"query": "OO단지 입주 후기", "display": 50} # 결과 컬럼: title, link, description, bloggername, postdate

    🎯마케팅 활용

    • 단지·브랜드 언급량 추이
    • 실거주 후기 톤(긍/부정)
    • 경쟁 단지 바이럴 비교

    🔁응용

    • 여러 키워드를 반복문으로
    • 날짜별 포스팅 수 집계
    • description으로 감정 분석
    Tip — 한 번에 최대 100건(display), 시작 위치(start)를 바꿔가며 페이지네이션하면 더 많이 수집.

    4-4네이버 쇼핑 데이터 수집

    상품명·가격·쇼핑몰을 모아 가격대·트렌드 분석

    Python 코드 — 쇼핑 검색 + 가격 통계url = "https://openapi.naver.com/v1/search/shop.json" params = {"query": "이사 청소", "display": 100} items = requests.get(url, headers=headers, params=params).json()["items"] df = pd.DataFrame(items) df["lprice"] = df["lprice"].astype(int) # 최저가 숫자화 print("평균가:", df["lprice"].mean()) print("가격대 분포:", df["lprice"].describe())
    ChatGPT에게네이버 쇼핑 API로 '이사 청소' 상품 100개를 수집해서 가격 평균·최저·최고와 가격대 분포 히스토그램을 그려줘. 상위 노출 쇼핑몰도 집계해줘.

    4-5API 오류 수정

    API 특유의 에러는 패턴이 정해져 있다 — 코드만 알면 즉시 해결

    코드의미해결
    401인증 실패Client ID/Secret 오타·공백 확인
    403권한 없음해당 API(검색) 사용 설정 여부 확인
    429호출 한도 초과잠시 후/내일, 호출 간격 늘리기
    400잘못된 요청query 인코딩·파라미터 철자 확인
    에러 해결 프롬프트네이버 검색 API를 호출했더니 [에러코드/메시지]가 나와. 원인과 수정 방법을 알려주고, 고친 전체 코드를 다시 줘. [코드/에러] (붙여넣기)
    Tip — 응답을 print(res.status_code, res.text)로 찍어보면 원인이 바로 보인다.

    4-6쇼핑 데이터 기반 소비자 니즈 · 인사이트

    수집한 데이터를 ChatGPT/Claude에 넘겨 ‘읽을 수 있는 전략’으로 전환

  • 수집 데이터(엑셀/표)를 AI에 업로드
  • 가격대·키워드·셀러 패턴 분석 요청
  • 소비자 니즈 → 마케팅 메시지로 번역
    • 인사이트 도출 프롬프트첨부한 네이버 쇼핑/블로그 수집 데이터를 분석해줘. 1) 가격대별 수요 구간과 '심리적 가격선' 2) 후기에 반복되는 니즈·불만 키워드 Top 10 3) 우리 분양 마케팅에 쓸 메시지 3가지(근거 포함) 4) 표와 차트로 정리
    핵심 전환 — 데이터 ‘수집’은 절반. 진짜 가치는 “그래서 무엇을 해야 하나”로 바꾸는 해석에 있다.

    나만의 업무 전용 챗봇 ① — Custom GPT 기본

    반복 요청을 ‘전용 봇’으로 박제 — 코딩 없이 만들고 외부 API까지 연결 (5-1 ~ 5-6)

    5-1 GPTs 개념·사례 5-2 기업정보 분석 GPT 5-3 부동산 리포트 GPT 5-4 Actions·외부 API 5-5 SerpAPI 스키마 5-6 실행·공유

    5-1GPTs 개념과 업무 도입 사례

    Custom GPT = 지침·자료·기능을 미리 세팅한 ‘나만의 ChatGPT’

    🤖무엇인가

    • 역할·말투·규칙을 고정
    • 참고자료(지식) 내장
    • 외부 API 연결(Actions) 가능

    💼업무 도입 사례

    • 제안서 작성 봇(우리 양식 학습)
    • 계약 검토 봇(규정 내장)
    • 실거래가 조회 봇(API 연결)
    일반 대화와 차이 — 매번 “너는 ○○야, 이 양식으로…”를 반복하지 않는다. 한 번 만들어두면 팀 전체가 같은 품질로 재사용·공유.
    전제 — GPTs 제작·Actions는 ChatGPT Plus 이상 필요. ChatGPT ▸ 좌측 ‘GPT 탐색’ ▸ + 만들기.

    5-2기업정보 분석 Custom GPT 생성

    첫 GPT — 자료 없이 ‘지침(Instructions)’만으로 만드는 기본형

  • GPT 만들기 ▸ Configure(구성) 탭
  • 이름·설명·아이콘 설정
  • Instructions에 역할·출력 형식 작성(아래)
  • 오른쪽 미리보기에서 테스트
    • Instructions 예시 (지침)# 역할 너는 기업 분석 전문 애널리스트다. # 입력 사용자가 회사명을 주면 다음을 분석한다. # 출력 형식 (항상 이 구조) 1) 한 줄 요약 2) 사업 영역 3) 강점·약점(표) 4) 최근 이슈 5) 협업/영업 관점 시사점 - 불확실한 정보는 "확인 필요"로 표시하고 추측은 명시한다. - 모든 답변은 한국어, 표·불릿 위주, 결론부터.
    Tip — 대화하듯 ‘Create’ 탭에서 만들 수도 있지만, Configure 탭에서 직접 지침을 쓰는 게 통제·재현에 유리.

    5-3부동산 실거주 종합 리포트 GPTs

    자료(지식 파일) 내장형 + 외부 검색 연동의 실전 GPT

    📚Knowledge(지식)

    • 입지 체크리스트 PDF
    • 학군·교통·생활권 기준표
    • 리포트 템플릿(우리 양식)

    🔗Capabilities/Actions

    • 웹 검색 ON
    • 외부 API(SerpAPI 등) 연결
    • 코드 실행(표·차트)
    Instructions 핵심부사용자가 '지역/단지'를 주면 실거주 관점 종합 리포트를 만든다. [항목] 입지·교통 / 학군 / 생활인프라 / 호재·리스크 / 종합점수(10점) - 내장 체크리스트(지식)를 기준으로 평가 - 최신 정보는 검색/Action으로 보강하고 출처 표기 - 마지막에 '실거주 추천 등급'과 한 줄 코멘트
    구성 요소 3종 — 지침(어떻게 행동), 지식(무엇을 근거로), Actions(어디서 데이터를). 다음 슬라이드에서 Actions 연결.

    5-4Actions와 외부 API 연결

    GPT가 스스로 외부 데이터를 가져오게 만드는 핵심 기능

    🧠원리

    GPT에게 ‘API 사용 설명서(스키마)’를 주면, 필요할 때 그 API를 호출해 실시간 데이터로 답한다

    🛠️설정 위치

    Configure ▸ Actions ▸ Create new action ▸ OpenAPI 스키마 입력 ▸ 인증(API Key) 입력

  • 연결할 API 선택(예: SerpAPI = 구글 검색 결과)
  • OpenAPI 형식 스키마 작성/붙여넣기(5-5)
  • Authentication에 API Key 등록
  • 미리보기에서 “GPT가 API를 호출하는지” 테스트
    • 안심 — 스키마도 ChatGPT에게 “이 API용 OpenAPI 3.1 스키마 만들어줘”라고 시키면 된다(다음 슬라이드).

    5-5SerpAPI 기반 스키마 생성

    검색 결과를 GPT가 가져오게 하는 OpenAPI 스키마 예시

    OpenAPI 3.1 스키마 (Actions에 붙여넣기)openapi: 3.1.0 info: title: SerpAPI Search version: "1.0" servers: - url: https://serpapi.com paths: /search: get: operationId: search parameters: - name: q in: query required: true schema: { type: string } - name: engine in: query schema: { type: string, default: google }
    ChatGPT에게 (스키마 위임)SerpAPI의 google 검색(/search, 파라미터 q, engine, api_key)을 GPT Actions에서 쓸 수 있는 OpenAPI 3.1 스키마로 만들어줘. 인증은 API Key(query)로.
    주의 — api_key는 스키마가 아니라 Authentication 칸에 넣어 노출을 막는다.

    5-6Custom GPT 실행 및 공유

    완성한 GPT를 테스트하고, 팀에 배포

  • 미리보기에서 실제 질문으로 검증(Action 호출 확인)
  • 우측 상단 Create/Update ▸ 저장
  • 공유 범위 선택: 나만 / 링크 가진 사람 / GPT Store 공개
  • 링크 복사 → 팀에 전달

    • ⚠️공유 전 점검

    • 지식 파일에 기밀 없는지
    • API Key 노출 안 되는지
    • 회사 정책상 외부 공개 가능한지

    ✅좋은 GPT의 조건

    • 지침이 구체적·일관
    • 출력 형식 고정
    • 모르면 ‘모른다’ 하도록
    모듈 5 정리 — 지침 + 지식 + Actions = 업무 전용 봇. 한 번 만들면 팀 전체의 생산성 자산이 된다.
    06

    나만의 업무 전용 챗봇 ② — 부동산 투자 시뮬레이터

    국토교통부 실거래가 API를 연결해, 질문하면 투자 분석을 내놓는 GPT (6-1 ~ 6-6)

    6-1 시뮬레이터 개요 6-2 실거래가 API 6-3 기본 설정 6-4 상세 지침 6-5 스키마 연결 6-6 분석 도출

    6-1부동산 투자 시뮬레이터 GPTs 생성

    “이 단지 어때?”라고 물으면 실거래 데이터 기반으로 답하는 봇

    🎯목표

    • 지역·단지 입력 → 실거래가 조회
    • 시세 흐름·수익률 시뮬레이션
    • 매수 의사결정 보조 리포트

    🧱구성

    • 지침: 분석 로직·출력 형식
    • Actions: 국토부 실거래가 API
    • 코드 실행: 수익률 계산·차트
    모듈 5와 차이 — 5는 ‘검색(SerpAPI)’, 6은 ‘정형 공공데이터(실거래가)’ 연결. 숫자 기반이라 계산·시뮬레이션이 핵심.
    전제 — 투자 ‘참고용’ 도구다. 출력에 면책 문구를 넣고, 최종 판단은 사람이.

    6-2국토교통부 실거래가 API 기반 데이터 처리

    공공데이터포털(data.go.kr)의 아파트 매매 실거래가 API

  • data.go.kr 가입 ▸ ‘아파트 매매 실거래가’ 검색
  • 활용 신청 ▸ 서비스키 발급
  • 지역코드(법정동 5자리) + 계약월(YYYYMM)로 조회
    • 요청 예시 — 강남구(11680) 2026년 4월GET https://apis.data.go.kr/.../getRTMSDataSvcAptTrade ?serviceKey=발급키 &LAWD_CD=11680 # 지역(법정동) 코드 &DEAL_YMD=202604 # 계약 연월 # 응답(XML): 아파트명, 전용면적, 거래금액, 층, 건축년도 …
    ChatGPT에게공공데이터포털 아파트 매매 실거래가 API로 강남구(11680) 2026년 4월 거래를 조회해서 아파트명·면적·금액·층을 표로 정리하는 파이썬 코드를 만들어줘. 응답이 XML이니 파싱도 포함해서.

    6-3GPTs 기본 설정

    이름·설명·시작 질문(Conversation starters)으로 사용성 세팅

    🏷️기본 정보

    • 이름: ‘부동산 투자 시뮬레이터’
    • 설명: 실거래 기반 분석 봇
    • 아이콘: 자동 생성 가능

    💬시작 질문 예

    • “강남구 84㎡ 최근 시세는?”
    • “이 단지 3년 가격 흐름 보여줘”
    • “전세 끼고 매수 수익률 계산”

    ⚙️Capabilities

    Code Interpreter ON(수익률 계산·차트), 필요 시 웹 검색 ON. 지식 파일로 ‘지역코드표·면책문구’ 첨부.

    6-4상세 지침 작성

    GPT의 ‘두뇌’ — 어떤 순서로 분석하고 무엇을 출력할지 규정

    Instructions (투자 시뮬레이터)# 역할 너는 부동산 투자 분석가다. 국토부 실거래가 Action으로 데이터를 가져와 분석한다. # 처리 순서 1. 사용자의 지역/단지/면적을 파악 (없으면 되묻는다) 2. 지역코드·계약월로 Action 호출 → 실거래 수집 3. Code로 평균/최고/최저, 면적당 단가, 월별 추세 계산 # 출력 형식 ① 시세 요약(표) ② 가격 추세(차트) ③ 면적당 단가 비교 ④ 간이 수익률 시뮬(매수가·전세가·보유기간 입력 시) ⑤ 리스크 체크 + ⚠️ "본 분석은 참고용" 면책 # 규칙 - 데이터 없으면 지어내지 말고 "거래 없음"으로 표시 - 숫자는 근거(거래건수)와 함께 제시
    핵심 — 지침이 곧 품질. ‘순서·형식·금지사항’을 구체적으로 박아둘수록 일관된 결과가 나온다.

    6-5API 스키마 연결 및 실행

    실거래가 API를 Actions에 등록하고 GPT가 직접 호출하게

    OpenAPI 스키마 (실거래가 Action)openapi: 3.1.0 info: { title: 실거래가, version: "1.0" } servers: [{ url: https://apis.data.go.kr/1613000/RTMSDataSvcAptTrade }] paths: /getRTMSDataSvcAptTrade: get: operationId: getAptTrade parameters: - { name: LAWD_CD, in: query, required: true, schema: { type: string } } - { name: DEAL_YMD, in: query, required: true, schema: { type: string } }
  • Actions ▸ 스키마 붙여넣기
  • Authentication ▸ API Key(serviceKey) 등록
  • 미리보기에서 “강남구 4월 거래” 질문 → 호출 확인

    • 6-6투자 분석 결과 도출 실습

    완성된 시뮬레이터로 실제 의사결정 시나리오 돌려보기

    실행 질문 예시강남구 OO아파트 84㎡의 최근 1년 실거래를 분석해줘. - 평균/최고/최저가와 면적당 단가 - 월별 가격 추세 차트 - 12억에 매수, 8억 전세 끼고 3년 보유 시 간이 수익률 - 매수 판단에 참고할 리스크 3가지

    WRAP-UP

    오늘 핵심 6가지

    • 엑셀 함수 — 함수명 암기 ✗, “데이터 구조 + 원하는 결과”를 AI에 설명 ○
    • VBA 매크로 — 반복 작업의 녹화 버튼. 코드는 AI가, 오류는 메시지째 되묻기
    • Python 크롤링 — Colab에서 설치 0, 수집→정리→분석 파이프라인 재사용
    • 네이버 API — 크롤링보다 안전한 공식 통로, 엔드포인트만 바꿔 확장
    • Custom GPT — 지침+지식+Actions로 업무 전용 봇 제작·공유
    • 투자 시뮬레이터 — 공공 실거래가 API 연결로 ‘데이터로 답하는’ GPT
    관통하는 원리 — 오늘 배운 모든 것은 “비개발자가 AI에게 시켜서 자동화한다”는 하나의 패턴. 도구만 다를 뿐이다.

    바로 쓰는 치트시트

    엑셀 함수 6선=IF(F2<=5,"마감임박","여유") =SUMIFS(E:E,C:C,"84A",D:D,"계약") =COUNTIFS(C:C,"84A",D:D,"계약") =AVERAGEIF(A:A,"101동",E:E) =VLOOKUP(H2,DB!A:E,5,FALSE) =IFERROR(INDEX(A:A,MATCH(H2,B:B,0)),"미배정")
