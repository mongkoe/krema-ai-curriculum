    PART 1 · 설치와 첫 영상
    01

    HyperFrames 개요

    코드로 영상을 ‘설계’하는 새로운 방식

    정의 작동 원리 전체 워크플로

    1-1HyperFrames란?

    HeyGen이 공개한 오픈소스 영상 렌더링 프레임워크

    🎞️정의

    HeyGen이 만든 오픈소스 영상 렌더링 프레임워크. 무료로 누구나 사용

    💡핵심 원리

    HTML/CSS/JS로 ‘영상 설계도’를 쓰면 → MP4로 렌더링

    발상의 전환 — 타임라인을 마우스로 편집(프리미어)하는 대신, ‘웹페이지 만들 듯 코드로 영상을 설계’한다. 8·9일차에 웹앱을 만든 그 방식이 영상 제작으로.
    왜 강력한가 — 코드라서 자동화·반복·데이터 연동이 쉽다. “100개 단지를 각각 다른 영상으로” 같은 대량 생성이 가능(노코드·MCP와 결합 여지).

    1-2작동 원리 — HTML/CSS/JS → MP4

    웹 기술로 ‘장면’을 그리면, 그게 프레임이 되어 영상으로

    HTML/CSS/JS 설계도→ 브라우저로 장면 렌더→ 프레임 캡처→ FFmpeg로 MP4 합성
    영상 설계도 개념 (HTML/CSS)<!-- 한 '장면'을 웹페이지처럼 설계 --> <div class="scene"> <h1 class="title">OO자이 분양 안내</h1> <p class="subtitle">역세권 · 남향 · 6월 분양</p> </div> /* CSS 애니메이션이 곧 영상의 움직임 */ .title { animation: fadeIn 1s; }
    핵심 — 우리가 8일차에 만든 ‘HTML 대시보드’를 떠올리면 된다. 같은 웹 기술로 ‘움직이는 장면’을 만들고, FFmpeg가 그걸 영상 파일(MP4)로 굽는다.

    1-3오늘의 전체 워크플로

    설치 → 영상 → 음성 → 병합 → 제출

    ① 설치

    FFmpeg·HyperFrames

    ② 영상

    설계도→렌더링

    ③ 음성

    ElevenLabs

    ④ 병합

    영상+나레이션

    ⑤ 제출

    MP4·드라이브

    FFmpeg 설치→ HyperFrames 설치→ 영상 생성→ ElevenLabs 음성→ 병합·MP4·제출
    섹션 1 정리 — HyperFrames = ‘코드로 쓰는 영상’. 원리(HTML→MP4)를 알았으니, 이제 엔진(FFmpeg)부터 설치한다.
    PART 1 · 설치와 첫 영상
    02

    FFmpeg 설치

    MP4 렌더링 엔진 — 영상·음성 처리의 표준 도구

    FFmpeg란 설치

    2-1FFmpeg란 + 설치

    영상·음성을 자르고·붙이고·변환하는 ‘만능 미디어 엔진’

    ⚙️FFmpeg

    • 영상·음성 처리 표준(무료·오픈소스)
    • 프레임 → MP4 합성
    • 영상+음성 병합(섹션 6)

    🔗HyperFrames와

    HyperFrames가 만든 장면을 FFmpeg가 MP4로 굽는다. 영상 제작의 ‘엔진’

    설치 (Claude Code가 도와줌)# Windows (winget) / Mac (brew) — 환경에 맞게 winget install ffmpeg brew install ffmpeg # 설치 확인 ffmpeg -version
    실전 — 설치 명령을 외울 필요 없다. Claude Code에 “내 OS에 FFmpeg 설치해줘”라고 하면 환경을 확인해 알맞게 설치·검증한다.
    PART 1 · 설치와 첫 영상
    03

    HyperFrames 설치

    Claude Code로 직접 설치

    설치 방식 확인

    3-1Claude Code로 직접 설치

    ‘빈 폴더 + 저장소 + 확인 후 설치’ (9일차 원칙 동일)

  • 새 빈 폴더에서 시작
  • HyperFrames 저장소를 Claude Code에 전달
  • “무엇인지 확인·설명 후 설치” 요청
  • 의존성(Node 패키지) 설치·확인
    • Claude Code 프롬프트HyperFrames(HeyGen 오픈소스 영상 프레임워크)를 이 빈 폴더에 설치하고 싶어. 먼저 구조와 필요 조건을 설명한 뒤, FFmpeg가 연동되는지 확인하고, 단계별로 설치·실행 준비를 도와줘.
    안전 — 외부 코드 설치는 ‘무엇을 하는지’ 확인 후. 의존성·실행 스크립트를 Claude가 설명하게 한 뒤 진행.
    섹션 3 정리 — FFmpeg(엔진) + HyperFrames(프레임워크) 준비 완료. 드디어 첫 영상을 만든다.
    PART 1 · 설치와 첫 영상
    04

    영상 실습 — 첫 렌더링

    최신 이슈를 온라인 서치해 영상으로

    서치 → 설계도 브라우저 표시 → 렌더링

    4-1최신 이슈 서치 → 영상 설계도

    ‘무엇을 담을지’를 AI가 조사하고, 영상 장면으로 구성

    온라인 서치→ 대본·장면 구성→ HTML 설계도 생성
    영상 생성 프롬프트'2026 수도권 분양 시장 핵심 이슈'를 온라인에서 조사해서, 30초 분량 정보 영상으로 만들어줘. - 도입(타이틀) → 핵심 3가지(장면별 텍스트·차트) → 마무리(CTA) - HyperFrames 설계도(HTML/CSS), 깔끔한 모션, 브랜드 톤(다크+레드)
    통합의 정점 — 5일차(리서치)·3일차(영상 기획)·8일차(코드)가 한 흐름으로. 조사 → 대본 → 영상 설계가 대화 한 번에.

    4-2브라우저 표시 → 렌더링

    장면을 브라우저로 미리 보고, MP4로 굽는다

    브라우저 미리보기→ 수정·확인→ FFmpeg 렌더링→ MP4
  • 설계도를 브라우저에서 장면으로 확인(움직임·타이밍)
  • 어색하면 “이 장면 1초 더 길게”처럼 대화로 수정
  • 완성되면 렌더링 명령 → 프레임을 MP4로 합성
    • 핵심 — 8·9일차의 ‘로컬 실행 → 캡처 피드백 → 개선’ 루프가 영상에도 그대로. 보면서 다듬는다. 다음 PART에서 목소리를 입힌다.
    PART 2 · 나레이션
    05

    ElevenLabs — 한국어 보이스와 Voice ID

    텍스트를 자연스러운 ‘사람 목소리’로

    한국어 음성 합성 Voice ID

    5-1한국어 음성 합성

    대본을 넣으면 자연스러운 한국어 나레이션이 나온다

    🎙️ElevenLabs

    • 고품질 한국어 TTS(음성 합성)
    • 감정·속도·억양 조절
    • 광고·영상 나레이션에 최적

    🎯분양 영상 활용

    • 단지 소개 나레이션
    • 모델하우스 도슨트(3·5일차)
    • 숏폼 보이스오버
    나레이션 스크립트 예"강남 역세권, 전 세대 남향 설계. 지금 OO자이가 여러분의 새로운 시작을 기다립니다. 6월, 특별한 기회를 놓치지 마세요." → 톤: 따뜻하고 신뢰감 있게, 약간 느린 속도
    팁 — 문장부호·줄바꿈으로 ‘쉬는 지점’을 조절. 숫자·영어는 발음이 어색할 수 있어 한글로 풀어쓰면 자연스럽다.

    5-2Voice ID — 일관된 목소리 고정

    여러 영상에 ‘같은 목소리’를 쓰려면 Voice ID로 지정

    🆔Voice ID란

    각 목소리의 고유 식별자. 이 ID로 항상 같은 보이스를 호출

    🎚️왜 중요

    시리즈·캠페인 영상의 ‘브랜드 보이스’ 일관성. 영상마다 목소리가 바뀌면 안 됨

    Voice ID로 음성 생성 (개념)# 선택한 한국어 보이스의 ID를 지정 voice_id = "21m00Tcm4TlvDq8ikWAM" text = "강남 역세권, 전 세대 남향 설계..." # → ElevenLabs API가 음성(mp3) 생성
    섹션 5 정리 — ElevenLabs로 한국어 나레이션을, Voice ID로 일관성을. 4일차 ‘프롬프트 자산화’처럼 ‘브랜드 보이스’도 자산. 이제 영상과 합친다.
    PART 2 · 나레이션
    06

    나레이션 병합

    스크립트 → 음성 → 병합 → MP4 → 제출

    병합 흐름 FFmpeg 병합 렌더·드라이브 제출

    6-1스크립트 → 음성 → 영상 병합

    영상(HyperFrames)과 음성(ElevenLabs)을 FFmpeg로 합친다

    스크립트→ ElevenLabs 음성(mp3)→ HyperFrames 영상(mp4)→ FFmpeg 병합
    FFmpeg 영상+음성 병합 명령# 영상(video.mp4)에 나레이션(voice.mp3)을 입힘 ffmpeg -i video.mp4 -i voice.mp3 \ -c:v copy -c:a aac -shortest \ final.mp4
    타이밍 맞추기 — 나레이션 길이에 맞춰 장면 시간을 조절(또는 그 반대). 안 맞으면 “음성 길이에 영상 맞춰줘”라고 Claude에 요청.

    6-2MP4 렌더링 → 구글 드라이브 제출

    최종 영상을 굽고, 공유·제출까지

  • 최종 MP4 렌더링(해상도·비율 설정: 16:9 / 9:16)
  • 미리보기로 음성·자막·타이밍 검수
  • 구글 드라이브 업로드
  • 공유 링크 생성 → 제출
    • 완성 — 코드로 영상을 설계하고, AI 목소리를 입혀, MP4로 구워 제출. 촬영·편집·성우 없이 기획자 혼자 ‘영상 한 편’을 완성했다.
    섹션 6 정리 — 스크립트 → 음성 → 병합 → MP4 → 제출. 이미지(3일차) → 영상화 → 나레이션까지, 영상 제작의 풀 사이클을 코드로 자동화.
